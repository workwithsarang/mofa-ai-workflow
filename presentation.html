The Problem

Why does this
project exist?

Modern AI systems need reusable, composable text-processing units that can be wired together without writing new code every time.

Pain Points
Hard-coded pipelines — change one step and everything breaks
No visibility into what each step produced
No user isolation — everyone shares the same agents
No API — no way to integrate with other systems
Our Solution
Named agents with explicit types and JWT-isolated ownership
Workflows that chain exactly 2 agents and log every step
Full REST API + Swagger UI for integration

Agents

The 4 Agent Types

🔠 UPPERCASE
Converts every character to uppercase.
Input: "hello world" → "HELLO WORLD"
🔢 WORD_COUNT
Counts the number of words in the text.
Input: "hello world" → "2"
🔄 REVERSE
Reverses every single character in the string.
Input: "hello" → "olleh"
✂️ TRIM
Removes leading and trailing whitespace.
Input: " hello " → "hello"

Workflows

How Workflows Work

A workflow chains exactly 2 agents. Output of Agent A becomes the input of Agent B.

Example: "Uppercase then Count"
INPUT
"the quick brown fox"
→
STEP 1
Uppercase Bot
UPPERCASE
→
STEP 2
Word Counter
WORD_COUNT
→
OUTPUT
"4"
Step 1 output
"THE QUICK BROWN FOX"
Final output
"4"  (4 words counted)

User Flow

How to Use It

1
Register & Log In
Create an account at the frontend URL. Your JWT token is stored in the browser.
2
Create Agents
Go to Agents → New Agent. Pick a name and type (UPPERCASE, WORD_COUNT, REVERSE, or TRIM).
3
Build a Workflow
Go to Workflows → New Workflow. Name it and pick Agent A + Agent B in order.
4
Run the Workflow
Click Run, type any input string, hit Execute. See each step's output and the final result.

Engineering

Challenges I Solved

Problem
Prisma migrations missing in git
Render ran prisma migrate deploy but migrations folder was never committed → deployment failed
Solution
Committed backend/prisma/migrations/ to git. Always commit migration files.
Problem
Vercel team-level SSO blocked API
All *.vercel.app URLs under the team were behind Vercel Authentication — bypass tokens didn't help.
Solution
Moved backend to Render entirely. Render gives a public URL with no SSO layer.
Problem
Frontend calling localhost in production
VITE_API_BASE_URL env var was not set in Vercel — build fell back to http://localhost:5000
Solution
Set VITE_API_BASE_URL in Vercel project settings and redeployed.
Problem
GitHub push rejected — email privacy
GitHub blocked the push because the commit author email was set to a real private email address.
Solution
Used GitHub's noreply email alias and amended the commit before pushing.
